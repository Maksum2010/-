# Пошук у ширину (Breadth-First Search, BFS)

## 1. Основи графів: вершини, ребра, шляхи

### Що таке граф?

Граф — це структура даних, яка складається з:

- **вершин (vertices)** — об'єктів або точок;
- **ребер (edges)** — зв'язків між вершинами.

Наприклад, у соціальній мережі:

- люди — це вершини;
- дружба між людьми — це ребра.

### Приклад графа

```text
A --- B
|     |
|     |
C --- D
```

Тут:

- вершини: A, B, C, D;
- ребра: A-B, A-C, B-D, C-D.

### Шлях у графі

Шлях — це послідовність вершин, через які можна перейти від однієї вершини до іншої.

Наприклад:

```text
A → B → D
```

Довжина шляху дорівнює кількості ребер у ньому.

---

## 2. Представлення графів у пам’яті

Для зберігання графів у програмі використовують кілька способів.

### Список суміжності

Найпоширеніший спосіб.

```python
graph = {
    "A": ["B", "C"],
    "B": ["A", "D"],
    "C": ["A", "D"],
    "D": ["B", "C"]
}
```

Переваги:

- займає мало пам'яті;
- зручний для обходу графа.

---

### Матриця суміжності

Таблиця, де:

- 1 означає наявність ребра;
- 0 означає його відсутність.

|   | A | B | C | D |
|---|---|---|---|---|
| A | 0 | 1 | 1 | 0 |
| B | 1 | 0 | 0 | 1 |
| C | 1 | 0 | 0 | 1 |
| D | 0 | 1 | 1 | 0 |

Переваги:

- швидка перевірка існування ребра.

Недоліки:

- потребує більше пам'яті.

---

## 3. BFS: алгоритм і його реалізація

### Що таке BFS?

**Breadth-First Search (Пошук у ширину)** — алгоритм обходу графа, який спочатку відвідує всі найближчі вершини, а потім переходить на наступний рівень.

Для роботи використовується структура даних **черга (Queue)**.

---

### Принцип роботи

Нехай маємо граф:

```text
      A
     / \
    B   C
   / \   \
  D   E   F
```

Якщо почати з вершини A, порядок обходу буде:

```text
A → B → C → D → E → F
```

Алгоритм:

1. Додаємо стартову вершину в чергу.
2. Виймаємо вершину з черги.
3. Відвідуємо всіх її сусідів.
4. Додаємо невідвіданих сусідів у чергу.
5. Повторюємо, поки черга не порожня.

---

### Реалізація BFS на Python

```python
from collections import deque

def bfs(graph, start):
    visited = set()
    queue = deque([start])

    visited.add(start)

    while queue:
        vertex = queue.popleft()
        print(vertex, end=" ")

        for neighbor in graph[vertex]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)

graph = {
    "A": ["B", "C"],
    "B": ["D", "E"],
    "C": ["F"],
    "D": [],
    "E": [],
    "F": []
}

bfs(graph, "A")
```

Результат:

```text
A B C D E F
```

---

### Складність алгоритму

Для графа з:

- V вершин;
- E ребер.

Часова складність:

```text
O(V + E)
```

Просторова складність:

```text
O(V)
```

## 4. Приклади використання BFS у житті

### 1. Пошук найкоротшого маршруту

Навігатори можуть використовувати BFS для пошуку найкоротшого шляху в незважених графах.

Приклад:

- станції метро;
- автобусні зупинки;
- дорожні перехрестя.

---

### 2. Соціальні мережі

Пошук друзів друзів.

Наприклад:

- ваші друзі — рівень 1;
- друзі ваших друзів — рівень 2;
- далі — рівень 3 і т.д.

---

### 3. Комп’ютерні мережі

Пошук пристроїв у локальній мережі.

BFS допомагає послідовно обходити всі вузли мережі та знаходити підключені пристрої.

---

### 4. Ігри та лабіринти

Алгоритм використовується для:

- пошуку виходу з лабіринту;
- побудови маршруту персонажа;
- навігації NPC.

---

### 5. Веб-краулери

Пошукові системи можуть обходити сторінки сайту рівень за рівнем:

1. Головна сторінка.
2. Посилання з головної.
3. Посилання з цих сторінок.
4. І так далі.

---

## Висновок

Пошук у ширину (BFS) — один із базових алгоритмів роботи з графами. Він обходить вершини рівень за рівнем, використовуючи чергу, та дозволяє знаходити найкоротші шляхи в незважених графах. BFS широко застосовується в навігації, соціальних мережах, комп'ютерних мережах, пошукових системах та іграх.
